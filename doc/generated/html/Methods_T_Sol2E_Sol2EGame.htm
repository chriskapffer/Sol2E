AddDomainSystem
Adds a class which implements the IDomainSystem to the list of domain systems. Use this to register your system to the game update loop.
ChangeWindowSize
Changes the game window from windowed mode to fullscreen or vice versa.

Draw
This is called when the game should draw itself. It was made sealed, because clients should not need override it. If you want to be able to use custom drawing create a subclass of AbstractGraphicsSystem and register it to the list of domain systems by calling AddDomainSystem(yourSystem).
(Overrides GameDraw(GameTime).)
Initialize
Allows the game to perform any initialization it needs to before starting to run. This is where it can query for any required services and load any non-graphic related content. Calling base.Initialize will enumerate through any components and initialize them as well.
(Overrides GameInitialize()()()().)
LoadContent
LoadContent will be called once per game and is the place to load all of your content.
(Overrides GameLoadContent()()()().)
RemoveDomainSystem
Removes a class which implements the IDomainSystem from the list of domain system. Use this to remove one of the default systems, in order to replace it with your own subclass, e.g.: game.RemoveDomainSystem(game.GraphicsSystem); game.AddDomainSystem(myCustomGraphicsSystemWhichCanRenderEvenMoreStuff);
UnloadContent
UnloadContent will be called once per game and is the place to unload all content.
(Overrides GameUnloadContent()()()().)
Update
Allows the game to run logic such as updating the world, checking for collisions, gathering input, and playing audio. It was made sealed, because clients should not need override it. If you want to be able to update your custom logic create a subclass of AbstractDomainSystem and register it to the list of domain systems by calling AddDomainSystem(yourSystem).
(Overrides GameUpdate(GameTime).)
UpdateDebugInfo
Updates debug information from current game state. The string 'DebugInfo' will internally only update every DebugInfoUpdateInterval.
