Sol2E Game Engine
Documentation of Sol2E Game Engine generated by Sandcastle


This is the Sol2E Game Engine, created as a student project by Christoph Kapffer at the HTW Berlin. The intention for creating this project was purely based on self education. It can be seen as an exercise of implementing a framework which is based on a component system. Architectural details are loosely based upon the concepts of the article Entity Systems are the future of MMOG development by Adam Martin on his blog T=Machine. The title of the engine "Sol2E" refers to a letter from from Sol LeWitt to Eva Hesse, whose content can be found here. The Engine is based upon Microsofts XNA Framework 4.0. Current build only supports Windows machines. Windows Phone or Xbox are not supported.

Namespaces
Namespace Description
Sol2E
Default namespace. It provides the Sol2EGame class, which sould be used as a base class to a new game build
Sol2E.Audio
The audio subsystem, which provides audio related components to use in a game, as well as the AudioSystem class, which handles the logic of all related components.
Sol2E.Common
This namespace contains classes which are commonly used around the project such as Transform, Movement or scripting related symbols.
Sol2E.Core
This is the engines back bone. It contains the definition of Scene, Entity, Component, AbstractDomainSystem and others. Scenes, entities and components are internally managed by the Database class.
Sol2E.Graphics
The graphics subsystem, which provides graphics related components to use in a game, as well as the GraphicsSystem class, which handles the logic of all related components.
Sol2E.Graphics.UI
A sub namespace of Sol2E.Graphics, which contains the definitions of UIElement, UIBLabel, UIButton and UIButtonEvent.
Sol2E.Input
The input subsystem, which provides the InputSystem class, which updates the input states of mouse and keyboard devices. It also handles the invocation of input dependet scripts.
Sol2E.Physics
The physics subsystem, which provides physic related components to use in a game, as well as the PhysicsSystem class, which handles the logic of all related components.
Sol2E.Utils
This provides utility classes which are used in almost all the other assemblies.
